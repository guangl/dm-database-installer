<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Shell pipe → CLI | `curl \| sh` 管道中 stdin 已被 curl 占用；CLI 必须不读 stdin |
| User args → clap | 命令行参数（--package, --checksum）是不可信输入 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Denial of Service | `install --defaults` 模式 stdin 读取 | mitigate | `--defaults` flag 解析为 bool；Plan 03 在 confirm_immutable_params 中以 `if skip { return Ok(()); }` 在任何 stdin 读取前短路（CONTEXT.md D-06, RESEARCH.md Pitfall 4） |
| T-01-02 | Tampering | `--package` 路径参数 | mitigate | clap 解析为 `PathBuf`；实际文件打开在 Plan 02 (`checksum.rs`) 和 Plan 03 (`package.rs`) 进行，路径将在那里 canonicalize |
| T-01-SC | Tampering | Cargo 依赖供应链 | mitigate | 所有 14 个依赖通过 crates.io 高下载量库（见 RESEARCH.md §Package Legitimacy Audit）；版本号锁定在 Cargo.toml + Cargo.lock 提交至 git |
</threat_model>
