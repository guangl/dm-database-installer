# 关键约束

- **Pitfall 1 (root 检测)**：在 install 入口（main.rs dispatch 前后均可，本 plan 选 main.rs 在 install 分支前）调用 root 检测；非 root 则用 `anyhow::bail!` 退出，提示 "请使用 sudo dm-installer install ..." 或 "请以 root 用户运行"；选用 `unsafe { libc::getuid() } == 0` 或 `std::env::var("USER").as_deref() == Ok("root")` 两条路径——本 plan 选 `unsafe { libc::geteuid() } == 0` 路径（不引入新 crate 不可行——`libc` 已被 reqwest/tokio 间接依赖但需在 Cargo.toml 显式声明）。**为避免引入新顶层 crate，使用 `std::env::var("USER")` + 检测 `/proc/self/status` UID 行的 fallback 模式，二者均不可用则保守通过并 tracing::warn!**。
- **Assumption A3**：`dm_service_installer.sh` 在 `{install_path}/script/root/dm_service_installer.sh`；运行时检测脚本存在，不存在则 anyhow::bail! 提示 DM 版本可能变更
- **Assumption A4 + Pitfall 5**：dmdba 用户可能未自动创建；本 plan 在调用 dm_service_installer.sh 前检测 `id dmdba` 是否成功，不成功则 `tracing::warn!` 但不阻止（DM 自带脚本通常会处理）
- 服务名：`DmService{instance_name}.service`，本 plan instance_name=DMSERVER → `DmServiceDMSERVER.service`
- 函数 < 40 行
- 函数命令构建抽出到 `build_service_installer_command` / `systemctl_enable_command` 便于单元测试

| Boundary | Description |
|----------|-------------|
| User → install 入口 | 非 root 用户尝试安装 → 必须 fail-fast |
