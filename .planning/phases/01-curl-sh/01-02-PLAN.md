<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `--config <toml>` → validate | 不可信 TOML 输入 → serde 反序列化 |
| `--package <iso>` → checksum | 不可信文件路径 → File::open + SHA-256 |
| `--checksum <hex>` → checksum | 不可信 hex 字符串 → 与计算结果字符串比较 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `--package` 路径遍历 (`../../etc/passwd`) | mitigate | Plan 03 `package::extract_dminstall_bin` 入口处 `Path::canonicalize()`；本 plan 仅做 SHA-256 校验（无解析路径），校验输入只是文件二进制，不影响系统 |
| T-02-02 | Tampering | SHA-256 校验绕过 (`--checksum` 不传) | accept | CONTEXT.md Open Question 1：DM 官网无公开 checksum 文件；产品决策为未提供时 `tracing::warn!` 跳过；用户责任 |
| T-02-03 | DoS | `--config` 指向大文件（GB 级） | accept | `read_to_string` 加载至内存；TOML 解析失败比 OOM 早；攻击者本地 CLI 工具无收益 |
| T-02-04 | Information Disclosure | TOML 解析错误暴露内部字段名 | accept | anyhow 错误链显示 toml::de::Error；提供给用户帮助定位错误，无敏感信息 |
</threat_model>
