7) `pub async fn run_dmasmcmd_init(install_path: &str, storage: &DscStorageConfig, dsc_conf_dir: &str, runner: &dyn CommandRunner) -> Result<()>`
   - 命令构造（heredoc 风格）：将以下 6 条 dmasmcmd 子命令拼接为一段以 `\n` 分隔的字符串：
     - `create dcrdisk '{storage.dcr_disk}' 'dcr'`
     - `create votedisk '{storage.vote_disk}' 'vote'`
     - `create asmdisk '{storage.log_disk}' 'LOG0'`
     - `create asmdisk '{storage.data_disk}' 'DATA0'`
     - `init dcrdisk '{storage.dcr_disk}' from '{dsc_conf_dir}/dmdcr_cfg.ini'`
     - `init votedisk '{storage.vote_disk}' from '{dsc_conf_dir}/dmdcr_cfg.ini'`
   - 最终命令：`printf '%s\n' "...所有子命令..." | {install_path}/bin/dmasmcmd`，install_path 经 shell_quote。注意：dmasmcmd 在 stdin pipe 模式下逐行读取
   - 报错时输出 stdout 全文供调试

8) `pub async fn run_dmasmtool_create_diskgroups(install_path: &str, dmdcr_ini_path: &str, runner: &dyn CommandRunner) -> Result<()>`
   - 命令：`printf '%s\n' "create diskgroup 'DMLOG' asmdisk '/dev/raw/LOG0'" "create diskgroup 'DMDATA' asmdisk '/dev/raw/DATA0'" | {install_path}/bin/dmasmtool DCR_INI={dmdcr_ini_path}`
   - 注意：dmasmtool 通过 ASM 磁盘名（LOG0/DATA0，即上一步 dmasmcmd 创建的 asmdisk 名）创建磁盘组，而非块设备路径。固定常量 `'/dev/raw/LOG0'` 与 `'/dev/raw/DATA0'` 在 dmasmcmd 中已注册的逻辑名，按 RESEARCH.md §"Example 2" 处理

9) `pub async fn run_dminit_shared(first_node: &NodeConfig, all_nodes: &[NodeConfig], dminit: &DminitConfig, oguid: u32, storage: &DscStorageConfig, runner: &dyn CommandRunner) -> Result<()>`

   - 生成 dminit.ini 内容：`generate_dminit_ini(all_nodes, dminit, oguid, storage)` (Plan 02 函数)
   - 上传到 `{dsc_conf_dir}/dminit.ini`
   - 执行命令：`{install_path}/bin/dminit control={dsc_conf_dir}/dminit.ini`
   - 不传统命令行参数（避免 Pitfall 4：通过 control file 让 dminit 处理 +DMDATA 路径）

10) `pub async fn distribute_config_dir(first_node_index: usize, other_node_index: usize, dminit: &DminitConfig, first_runner: &dyn CommandRunner, other_runner: &dyn CommandRunner) -> Result<()>`
    - 设 first_dir = `{dminit.data_path}/dsc{other_node_index}_config`（first_node 上 dminit 生成的其他节点 config）
    - 第一步：在 first_runner 上执行 `tar czf /tmp/dsc{other_node_index}_config.tar.gz -C {dminit.data_path} dsc{other_node_index}_config`
    - 第二步：sftp_read 读取 `/tmp/dsc{other_node_index}_config.tar.gz` 字节
    - 第三步：sftp_write 上传到 other_runner 的 `/tmp/dsc{other_node_index}_config.tar.gz`
    - 第四步：在 other_runner 上 `mkdir -p {dminit.data_path}` 然后 `tar xzf /tmp/dsc{other_node_index}_config.tar.gz -C {dminit.data_path}`
    - 所有路径经 shell_quote
    - 注意：first_node_index 参数本任务暂时只用于日志（tracing::info!("分发 dsc{i} 节点 config 目录")）；签名为后续扩展保留对称性

11) `pub async fn verify_dsc_node(node: &NodeConfig, dminit: &DminitConfig, runner: &dyn CommandRunner) -> Result<()>`

    - SQL：`SELECT STATUS$, MODE$ FROM V$INSTANCE;`
