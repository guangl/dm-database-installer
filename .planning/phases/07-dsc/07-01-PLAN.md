<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dsc.toml → ClusterSpecificConfig | 用户提供的磁盘路径字符串进入配置结构体；本计划只做结构校验，不发起 shell 调用 |
| 磁盘checkpoint JSON 文件 → ClusterCheckpoint | 本地文件反序列化；JSON 解析失败已有 try/catch 容错（loadfrom 第 49-54 行） |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | DscStorageConfig 字段值（dcr_disk 等可能包含 shell 元字符如 `; rm -rf /`） | mitigate | 本计划不发起 shell 调用；下一计划 (Plan 02/03) 在拼接命令前通过 `crate::common::shell_quote` 转义。本计划负责校验"非空 + 路径互异"避免误用 |
| T-07-02 | Information Disclosure | dsc.toml 中 `dsc_storage` 字段对外暴露磁盘路径 | accept | 路径本身非敏感信息；SSH 凭据已在 ssh credentials 段处理 |
| T-07-03 | Tampering | ClusterCheckpoint JSON 文件被外部修改后欺骗断点续传跳过未完成阶段 | accept | checkpoint 文件落在用户工作目录，与 dsc.toml 信任域相同；用户对该目录有完全控制权，攻击者可直接修改更高价值文件 |
</threat_model>
