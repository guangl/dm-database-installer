<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → crates.io | CI 中 cargo 下载依赖；用 Cargo.lock 锁版本 |
| 用户 → GitHub Releases asset | 通过 cargo-dist 生成的 installer.sh 下载预编译二进制；asset 完整性由 GitHub Releases 的 attestation + SHA256 manifest 保护 |
| CI runner → 构建产物 | release.yml 写入 GitHub Release；用 GITHUB_TOKEN 权限边界限定 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Tampering | release.yml 中 NASM 通过 chocolatey 安装（runner 环境注入风险） | accept | choco 官方包；windows-2022 runner 由 GitHub 维护；NASM 是固定版本依赖，被替换风险极低 |
| T-04-07 | DenialOfService | cargo-dist 0.32.0 自身的供应链；cargo install 拉取 | accept | cargo-dist 已在 CLAUDE.md 推荐且 axodotdev 官方维护，多个知名项目（uv, Rye）使用 |
| T-04-08 | InformationDisclosure | release.yml 中可能 echo GITHUB_TOKEN 到 log | mitigate | 使用 cargo-dist 生成的标准 release.yml；不要手动添加 echo $GITHUB_TOKEN；不要修改 dist plan/build/host 等 jobs |
| T-04-09 | Spoofing | x86_64-pc-windows-gnu 目标（D-04 原文）会因 ring/mio 不兼容产出无效或非预期二进制 | mitigate | Plan 02 决策：切换到 x86_64-pc-windows-msvc，在 windows-2022 runner 上原生构建，已被 russh CI 验证可行 |
| T-04-SC | Tampering | cargo-dist 0.32.0 安装（仅开发者机器 + CI） | accept | cargo-dist 在 CLAUDE.md 明确推荐；本 plan 不向 [dependencies] 添加运行时 crate，dist 是构建工具不会被打包进二进制 |
</threat_model>
