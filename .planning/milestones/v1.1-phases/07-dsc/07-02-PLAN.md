DSC 配置格式（RESEARCH.md §"DSC 配置文件结构"）：
- dmdcr_cfg.ini：3 个 [GRP] 段（CSS / ASM / DB），各段 DCR_GRP_N_EP = nodes.len()，每个 EP 含 host + port（CSS 用 9341/9343 序列、ASM 用 9349/9351 序列、DB 用 dminit.port + 1 序列）。顶部含 DCR_N_GRP = 3、DCR_VTD_PATH = vote_disk、DCR_OGUID = oguid
- dmasvrmal.ini：N 个 [MAL_INSTn] 段（n 从 0 起），每段含 MAL_INST_NAME = node.instance_name、MAL_HOST = node.host、MAL_PORT = ASM 端口（9349 + i*2）
- dmdcr.ini：每节点不同，含 DMDCR_PATH = dcr_disk、DMDCR_MAL_PATH = dmasvrmal_path、DMDCR_SEQNO = node_index、DMDCR_ASM_RESTART_INTERVAL = 60、DMDCR_DB_RESTART_INTERVAL = 60、DMDCR_ASM_STARTUP_CMD = `{install_path}/bin/dmasmsvr DCR_INI={dmdcr_ini_path}`、DMDCR_DB_STARTUP_CMD = `{install_path}/bin/dmserver {dm_ini_path} dcr_ini={dmdcr_ini_path}`
- dminit.ini：顶部含 SYSDBA_PWD = dminit.sysdba_password、DCR_PATH = dcr_disk、DCR_OGUID = oguid、DB_NAME = "GRP_DSC"、SYSTEM_PATH = "+DMDATA/data"；每节点一个 [DSCi] 段，含 CONFIG_PATH = `{data_path}/dsc{i}_config`、PORT_NUM = dminit.port + i、MAL_HOST = node.host、MAL_PORT = node.mal_port、LOG_PATH = `+DMLOG/log/dsc{i}_log01.log`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DscStorageConfig / NodeConfig 字段 → INI 字符串 | 用户配置最终写入 INI 文件；本计划只做字符串拼接，不直接发起 shell 调用 |
| 生成的 INI 内容 → 后续 SFTP 上传 | Plan 03 通过 sftp_write 上传，内容以字节传递，不经过 shell 解析 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | Tampering | NodeConfig.instance_name 含 INI 控制字符（如 `]` 或 `\n` 可能破坏 [DSCi] 段语义） | accept | instance_name 已在 Phase 1 配置验证中限制为标识符语义；DSC 内部沿用 dsc0/dsc1 序号，instance_name 仅作 MAL_INST_NAME 字面值，最坏情况导致配置无效而非命令注入 |
| T-07-05 | Tampering | DscStorageConfig.dcr_disk 等含 INI 注入字符 | mitigate | Plan 03 在 SFTP 上传后的 dmasmcmd 命令拼接时使用 shell_quote 包裹；本计划不直接拼接 shell 命令 |
| T-07-06 | Information Disclosure | dminit.ini 含 SYSDBA_PWD 明文 | accept | DM 静默安装原生设计如此，运行后该文件应只对 dmdba 用户可读（Plan 03 设置权限）；密码本身已由用户在 dsc.toml 中提供 |
</threat_model>
